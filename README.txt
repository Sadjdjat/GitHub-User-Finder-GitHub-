# GitHub User Finder

## Автор
**Керимов Саджджат **

## Описание
**GitHub User Finder** - это графическое приложение для поиска пользователей GitHub с возможностью сохранения избранных аккаунтов. Приложение использует официальное GitHub API и предоставляет интуитивно понятный интерфейс для работы с данными пользователей.

### Основные возможности:
- 🔍 Поиск пользователей GitHub по username
- 📊 Отображение детальной информации (ID, имя, тип аккаунта, количество репозиториев, подписчиков)
- ⭐ Добавление пользователей в избранное
- 💾 Автоматическое сохранение избранных пользователей в JSON файл
- 🗑 Удаление пользователей из избранного (по одному или всех)
- 📅 Отслеживание даты добавления в избранное
- ✅ Валидация ввода (поле поиска не может быть пустым)
- 🎨 Удобный графический интерфейс с вкладками

## Технологии
- **Python 3.7+** - язык программирования
- **Tkinter** - библиотека для создания GUI
- **urllib** - встроенный модуль для HTTP-запросов
- **JSON** - формат хранения данных
- **Git** - система контроля версий

1. Клонируйте репозиторий:
```bash
git clone https://github.com/your-username/GitHubUserFinder.git
cd GitHubUserFinder
Запустите приложение:

Примечание: Для работы приложения не требуется установка дополнительных библиотек, так как используются только встроенные модули Python.

Как использовать GitHub API
Эндпоинты API
Приложение использует следующий эндпоинт GitHub API:

text
GET https://api.github.com/users/{username}
Параметры запроса
{username} - имя пользователя GitHub (обязательный параметр)

Заголовки запроса
User-Agent - обязательный заголовок для всех запросов к GitHub API

Accept: application/vnd.github.v3+json - формат ответа JSON
Пример ответа API
json
{
  "login": "octocat",
  "id": 1,
  "avatar_url": "https://github.com/images/error/octocat_happy.gif",
  "type": "User",
  "name": "The Octocat",
  "company": "@github",
  "blog": "https://github.blog",
  "location": "San Francisco",
  "email": null,
  "bio": null,
  "public_repos": 8,
  "followers": 100,
  "following": 0,
  "created_at": "2011-01-25T18:44:36Z",
  "updated_at": "2023-12-01T12:00:00Z"
}
Коды ответов API
Код	Значение	Описание
200	OK	Успешный запрос, пользователь найден
404	Not Found	Пользователь не существует
403	Forbidden	Превышен лимит запросов
500	Internal Server Error	Ошибка на стороне GitHub
Лимиты API
Без авторизации: 60 запросов в час

С авторизацией (токен): 5000 запросов в час

Примеры использования
Тест 1: Поиск существующего пользователя
Действия:

Введите octocat в поле поиска

Нажмите кнопку "Найти" или клавишу Enter

Ожидаемый результат:

В таблице результатов появится строка с данными пользователя

Статусная строка покажет "Пользователь 'octocat' найден"

Тест 2: Поиск несуществующего пользователя
Действия:

Введите nonexistentuser123456789 в поле поиска

Нажмите "Найти"

Ожидаемый результат:

Появится сообщение об ошибке "Пользователь не найден"

Статусная строка покажет ошибку

Тест 3: Проверка валидации ввода
Действия:

Оставьте поле поиска пустым

Нажмите "Найти"

Ожидаемый результат:

Появится предупреждение "Поле поиска не может быть пустым!"

Тест 4: Добавление пользователя в избранное
Действия:

Найдите пользователя (например, octocat)

Дважды кликните по строке с результатом

Ожидаемый результат:

Появится сообщение об успешном добавлении

Статус в колонке "Действия" изменится на "✓ В избранном"

Строка в результатах поиска подсветится зеленым

Пользователь появится на вкладке "Избранное"

Тест 5: Удаление из избранного
Действия:

Перейдите на вкладку "Избранное"

Кликните правой кнопкой мыши по пользователю

Выберите "Удалить из избранного"

Ожидаемый результат:

Пользователь исчезнет из списка избранного

Статус в результатах поиска изменится на "⭐ Добавить"

Тест 6: Сохранение данных при закрытии
Действия:

Добавьте нескольких пользователей в избранное

Закройте приложение

Откройте файл favorites.json

Ожидаемый результат:

Файл favorites.json создан

Файл содержит JSON-массив с данными всех избранных пользователей

Тест 7: Загрузка избранного при запуске
Действия:

Убедитесь, что файл favorites.json существует и содержит данные

Запустите приложение заново

Ожидаемый результат:

Избранные пользователи автоматически отобразятся на вкладке "Избранное"

Статусная строка покажет количество избранных пользователей

Тест 8: Массовое удаление из избранного
Действия:

Добавьте несколько пользователей в избранное

На вкладке "Избранное" нажмите кнопку "Удалить всех из избранного"

Подтвердите действие

Ожидаемый результат:

Все пользователи удалены из избранного

Файл favorites.json содержит пустой массив

Структура JSON файла favorites.json
json
[
  {
    "login": "octocat",
    "id": 1,
    "name": "The Octocat",
    "type": "User",
    "public_repos": 8,
    "followers": 100,
    "date_added": "2024-01-15 14:30:25",
    "avatar_url": "https://github.com/images/error/octocat_happy.gif",
    "html_url": "https://github.com/octocat"
  }
]
